Shell Scripting Guide Minimizing And Raising Windows For Internet Speed Testing

Jul 31, 2025 by ADMIN 80 views

Hey guys! Let's dive into how you can create a shell script that minimizes a terminal window, runs an internet speed test, and then brings the terminal back up with the results. This is super handy for those times you want to run a quick speed check without the terminal hogging your screen. We'll be using xdotool, a powerful command-line tool for X11, to make this happen. So, buckle up, and let's get started!

Understanding the Goal

So, what's the big picture here? We want a script that does the following:

  1. Minimize the Terminal Window: This gets the terminal out of our way while the speed test runs.
  2. Run the Internet Speed Test: We'll use a command-line tool like speedtest-cli for this.
  3. Raise the Terminal Window: Once the test is complete, we bring the terminal back into view so we can see the results.

This is a neat way to automate tasks and keep your workspace tidy. It's like having a little assistant that handles the boring stuff for you. Plus, it's a great way to learn more about shell scripting and window management in Linux.

Prerequisites: Installing xdotool and speedtest-cli

Before we jump into the script, we need to make sure we have the right tools installed. xdotool is our main player for window management, and speedtest-cli is a popular choice for running speed tests from the command line. If you don't have these installed, no worries! Here’s how you can get them:

Installing xdotool

xdotool is a command-line utility that allows you to simulate keyboard input and mouse actions, move and resize windows, and more. It's a must-have for any Linux power user. To install it, open your terminal and use your distribution's package manager.

For Debian/Ubuntu-based systems, run:

sudo apt-get update
sudo apt-get install xdotool

For Fedora/CentOS/RHEL systems, run:

sudo dnf install xdotool

For Arch Linux systems, run:

sudo pacman -S xdotool

Once the installation is complete, you can verify it by typing xdotool --version in your terminal. If it's installed correctly, you should see the version number.

Installing speedtest-cli

speedtest-cli is a command-line interface for testing internet bandwidth using speedtest.net. It’s lightweight and easy to use, making it perfect for our script. There are a couple of ways to install it:

Using pip (Python Package Installer):

If you have Python and pip installed, this is the recommended method. Run:

sudo pip install speedtest-cli

If you encounter permission issues, you might need to use --user flag:

pip install speedtest-cli --user

Make sure that the pip binaries are in your $PATH. You might need to add ~/.local/bin to your $PATH in your .bashrc or .zshrc file.

Using your distribution's package manager:

Some distributions have speedtest-cli in their repositories. For example, on Debian/Ubuntu, you can try:

sudo apt-get install speedtest-cli

However, the version in the repositories might not be the latest. Using pip is generally the best way to ensure you have the most up-to-date version.

To verify the installation, run speedtest-cli --version. You should see the version number if it’s installed correctly.

With xdotool and speedtest-cli installed, we're ready to move on to the script itself. These tools are the backbone of our operation, allowing us to control the terminal window and run the speed test seamlessly. Make sure you have them set up correctly before proceeding!

Crafting the Shell Script

Alright, let's get our hands dirty and write the shell script. We'll break it down step by step so you can see exactly what's going on. Here’s the script we’re aiming for:

#!/bin/bash

# Get the window ID of the active terminal
terminal_id=$(xdotool getactivewindow)

# Minimize the terminal window
xdotool windowminimize $terminal_id

# Run the internet speed test and store the results
results=$(speedtest-cli)

# Display the results (optional, can be logged instead)
echo "Speedtest Results:"
echo "$results"

# Raise the terminal window
xdotool windowraise $terminal_id

# Optional: Display a notification (requires notify-send)
if command -v notify-send &> /dev/null;
then
 notify-send "Speedtest Complete" "Download/Upload speeds are in the terminal."
fi

exit 0

Now, let's dissect this script piece by piece:

Shebang and Initial Setup

#!/bin/bash

This line, known as the shebang, tells the system to use /bin/bash to execute the script. It's the standard way to start a bash script. Think of it as the script's way of introducing itself to the system.

Getting the Terminal Window ID

terminal_id=$(xdotool getactivewindow)

Here, we're using xdotool getactivewindow to get the ID of the currently active window, which should be our terminal. The $(...) syntax is command substitution, which means the output of the command inside the parentheses is assigned to the variable terminal_id. This is crucial because we need to tell xdotool which window to minimize and raise.

Minimizing the Terminal Window

xdotool windowminimize $terminal_id

This line is where the magic happens. We're using xdotool windowminimize along with the $terminal_id we just obtained to minimize the terminal window. It’s like clicking the minimize button, but from the command line. This keeps the terminal out of sight while the speed test runs in the background.

Running the Internet Speed Test

results=$(speedtest-cli)

This line executes the speedtest-cli command, which performs the internet speed test. The output of the command, which includes the download speed, upload speed, and other details, is stored in the results variable. This variable will be used later to display the results.

Displaying the Results

echo "Speedtest Results:"
echo "$results"

These lines simply print the results of the speed test to the terminal. The first echo command prints a header, and the second echo command prints the contents of the results variable. This ensures that the user can see the speed test results once the terminal is raised.

Raising the Terminal Window

xdotool windowraise $terminal_id
