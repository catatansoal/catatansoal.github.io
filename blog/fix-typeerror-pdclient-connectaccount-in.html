Fix TypeError: PdClient.connectAccount In Pipedream SDK V2

Aug 5, 2025 by ADMIN 59 views

Hey guys, let's dive into a tricky issue some of you might be facing with Pipedream's SDK v2. Specifically, the dreaded TypeError: pdClient.connectAccount is not a function. This usually pops up when trying to use the Pipedream Connect flow in a Next.js client component after migrating from @pipedream/sdk v1.x to v2.0.0-rc.4. Let's break down what's happening and how to fix it.

Understanding the Problem

So, you've updated to Pipedream SDK v2, and suddenly your client-side code is throwing a TypeError when you try to call pdClient.connectAccount. This usually indicates that the method connectAccount isn't available on the PipedreamClient instance you're creating in the browser. Following the migration guide, you initialized PipedreamClient in the browser. Providing projectId, clientId, and clientSecret gets past the initial "Project ID is required" error, but then you hit this TypeError. The problem here is that the client-side instance created from the main PipedreamClient class doesn't include browser-specific methods like connectAccount. Moreover, initializing the client this way exposes the clientSecret in the browser, a significant security risk, unlike the v1 createFrontendClient pattern that only needed a temporary token. The core issue is that the standard PipedreamClient isn't designed for direct browser use and lacks the necessary browser-specific methods.

To make matters worse, this new initialization method requires you to expose your clientSecret in the browser – yikes! That's a major no-no from a security standpoint. The older createFrontendClient method in v1 only needed a temporary token, which was much safer. So, what's going on? Basically, the PipedreamClient you're instantiating directly isn't meant to be used client-side and doesn't have the connectAccount method.

Reproducing the Error

To reproduce this issue, you'll need:

  1. @pipedream/sdk version 2.0.0-rc.4
  2. A Next.js (v15+) application with the App Router.

Here's how to trigger the error:

  1. Create a Connect Token: In a server action, generate a connect_token for a user.
  2. Client Component Setup: In a client component ("use client"), create a function to handle account connections.
  3. Fetch and Initialize: Inside this function, fetch the connect_token and then initialize a new PipedreamClient instance, providing all credentials, including clientId and clientSecret.
  4. Call connectAccount: Attempt to call the .connectAccount() method on the newly created client instance.

Here’s some code that might cause you trouble:

// In a Next.js client component
import { PipedreamClient } from "@pipedream/sdk";
import { createConnectToken } from "@/app/actions/pipedream"; // Server action

// ...

const handleConnectAccount = async (appSlug: string) => {
  try {
    const tokenData = await createConnectToken(externalUserId);
    if (!tokenData.token) {
      console.error('Failed to create connect token');
      return;
    }
    
    // This initialization requires exposing secrets to the browser.
    const pdClient = new PipedreamClient({
      projectEnvironment: "development",
      clientId: process.env.NEXT_PUBLIC_PIPEDREAM_CLIENT_ID!,
      clientSecret: process.env.NEXT_PUBLIC_PIPEDREAM_CLIENT_SECRET!, // Insecure
      projectId: process.env.NEXT_PUBLIC_PIPEDream_PROJECT_ID!,
      externalUserId: externalUserId,
      tokenCallback: async () => tokenData
    });

    // The following line throws the TypeError
    await pdClient.connectAccount({
      app: appSlug,
      token: tokenData.token,
      onSuccess: ({ id: accountId }) => {
        console.log(`Account connected: ${accountId}`);
      },
      onError: (error) => console.error('Connection error:', error),
    });

  } catch (error) {
    console.error('Error initiating account connection:', error);
  }
};

The error you'll likely see is:

Error initiating account connection: TypeError: pdClient.connectAccount is not a function
    at handleConnectAccount

Ideally, pdClient.connectAccount() should be available on the client-side instance, triggering the Pipedream Connect authentication flow in a pop-up window. Also, client-side initialization shouldn't need the clientSecret.

Environment Details

  • @pipedream/sdk: ^2.0.0-rc.4
  • next: 15.4.5
  • react: 19.1.0
  • Framework: Next.js with App Router / Turbopack
  • Environment: Browser (Next.js Client Component)

Solution

The key to resolving this issue lies in using the correct client initialization method for browser environments. The standard PipedreamClient is designed for server-side use, while the @pipedream/sdk/browser module provides the BrowserClient class and the createFrontendClient function, which are specifically tailored for client-side interactions. The BrowserClient and createFrontendClient are designed to be used within browser environments, providing the necessary methods and security considerations for client-side operations. These tools handle the complexities of browser-based authentication and account connections without exposing sensitive credentials.

Instead of directly instantiating PipedreamClient with the clientSecret, you should use createFrontendClient (or BrowserClient) from @pipedream/sdk/browser. This method is designed for the browser and doesn't require the clientSecret. Here’s how you can modify your code:

import { createFrontendClient } from "@pipedream/sdk/browser";
import { createConnectToken } from "@/app/actions/pipedream";

const handleConnectAccount = async (appSlug: string) => {
  try {
    const tokenData = await createConnectToken(externalUserId);
    if (!tokenData.token) {
      console.error('Failed to create connect token');
      return;
    }

    const pdClient = createFrontendClient({
      projectId: process.env.NEXT_PUBLIC_PIPEDREAM_PROJECT_ID!,
      clientId: process.env.NEXT_PUBLIC_PIPEDREAM_CLIENT_ID!,
      token: tokenData.token, // Pass the token here
      projectEnvironment: "development",
    });

    await pdClient.connectAccount({
      app: appSlug,
      token: tokenData.token,
      onSuccess: ({ id: accountId }) => {
        console.log(`Account connected: ${accountId}`);
      },
      onError: (error) => console.error('Connection error:', error),
    });

  } catch (error) {
    console.error('Error initiating account connection:', error);
  }
};

Explanation:

  1. Import createFrontendClient: Import the correct method from @pipedream/sdk/browser.
  2. Use createFrontendClient: Initialize the client using createFrontendClient, passing the projectId and clientId. Do not pass the clientSecret here.
  3. Pass the Token: Ensure you're passing the token you received from your server action to the createFrontendClient configuration.

Key Takeaways

  • Use createFrontendClient or BrowserClient in the browser: These are designed for client-side use and handle security appropriately.
  • Don't expose clientSecret: Never include your clientSecret in client-side code.
  • Pass the token: Make sure to pass the connect token obtained from your server to the createFrontendClient configuration.

By following these steps, you should be able to resolve the TypeError and securely implement the Pipedream Connect flow in your Next.js application.
