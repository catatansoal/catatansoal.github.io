Origin Check Issues In Review Environment: A Deep Dive

Aug 1, 2025 by ADMIN 55 views

Hey everyone! Recently, some updates using resources under /shared-assets were merged into content, but they were failing on the stage and production environments. An example of this is this image.

The Issue and Its Resolution

The problem was that these assets weren't playing nicely with the origin checks in our cloud functions. But don't worry, the issue has been resolved! A fix was implemented in this pull request to allow both main and interactive examples origins to access these assets through the cloud function's proxy. Basically, we whitelisted the origins so everything could load smoothly.

Why Didn't We See This Earlier?

You might be wondering, "Why didn't we catch this sooner?" Well, the review environment has WILDCARD_ENABLED set to true. This setting essentially turns off all origin checks in the cloud function. It's like having a free pass where everything is allowed, which is great for quick testing but not so great for catching real-world issues.

The Importance of Environment Parity

For a smoother and more reliable development process, it's super important that our review environment closely mirrors our production and staging environments. This means that if something works in review, it should also work in the other environments (and vice versa). If the environments have different configurations, it can lead to unexpected issues and headaches down the road.

Locking Down Origin Checks

To achieve better parity, it would be really helpful to lock down the origin checks in the review environment too. We could do this by setting a specific origin, maybe something like *.review.mdn.allizom.org as the main origin. This way, we can ensure that the review environment behaves more like production and staging, catching potential problems before they make it to the live site.

By implementing stricter origin checks in the review environment, we can prevent those frustrating moments where things work in review but break in production. It's all about making our development workflow more reliable and predictable. This approach helps us maintain the quality and consistency of our content across all environments.

Specific Example: CSS Anchor Positioning

Let's dive into a specific example where this issue manifested. The problem arose in the context of CSS anchor positioning, specifically when trying to use the position-area property. The documentation for this feature can be found here.

The URL in Question

The specific URL where the issue was observed is: https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_anchor_positioning/Using#setting_a_position-area. This page demonstrates how to use the position-area property in CSS, and it relies on shared assets to display certain diagrams and examples.
