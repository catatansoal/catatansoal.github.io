Install Windows Server 2019 RDS CALs: A Step-by-Step Guide

Aug 12, 2025 by ADMIN 59 views

Hey guys! Ever wondered how to get Remote Desktop Services (RDS) up and running on your Windows Server 2019? Well, you're in luck! This guide will walk you through the process of installing and configuring Remote Desktop Services Client Access Licenses (CALs). We'll cover everything from the initial setup to activating those licenses, ensuring your users can connect seamlessly. Let's dive in and make your server a remote access powerhouse!

What are Remote Desktop Services and CALs?

Alright, before we jump into the nitty-gritty, let's break down what Remote Desktop Services and CALs are all about. Imagine RDS as the key that unlocks the door to your server. It allows users to connect to your server remotely and use applications, access files, and essentially experience a desktop environment as if they were sitting right in front of the server. Pretty cool, right?

Now, what about CALs? Think of them as the tickets that users need to get through that door. CALs (Client Access Licenses) are licenses that grant users the right to access the RDS functionality. Without the appropriate CALs, your users won't be able to connect remotely. There are two main types of CALs: User CALs and Device CALs. User CALs allow a specific user to access the server from any device, while Device CALs allow any user to access the server from a specific device. Choosing the right type depends on your organization's needs.

In essence, RDS enables remote access, and CALs ensure you're legally covered and provide the necessary access rights for your users. We're going to walk you through the installation process, ensuring you have the right pieces in place to remotely access your Windows Server 2019 environment.

Prerequisites Before You Start

Okay, before we get our hands dirty, let's make sure we have everything we need. Before installing RDS, there are a few things you'll need to have in place. First off, you'll need a Windows Server 2019 server, obviously! Make sure it's up and running and that you have administrative access. You'll also need an internet connection on the server, since we'll need to activate the RDS license server. Another important step is to ensure your server has a static IP address. This will help with stability and ensure your users can connect reliably.

Additionally, you'll need to have purchased the necessary RDS CALs for your users or devices. You can purchase these licenses through Microsoft or a reseller. Keep those license keys handy, as you'll need them during the activation process. Also, be sure that the server meets the minimum hardware requirements for RDS, including sufficient RAM and storage. It's best to check the official Microsoft documentation for the precise specifications. Also, it is crucial to verify the server's time and date settings, as incorrect settings can potentially cause issues during activation. Make sure that your server is properly configured for the local network as well.

Finally, it is useful to have a basic understanding of Windows Server administration and the Remote Desktop Services role. But don't worry if you're a beginner ‚Äì we'll guide you through each step. Once you have these prerequisites sorted, you're all set to start the RDS installation process.

Installing the Remote Desktop Services Role

Alright, time to get down to business and install the Remote Desktop Services role. This is the foundation for enabling remote access to your server. First, log into your Windows Server 2019 server with an administrator account. After you've logged in, open Server Manager. You can usually find it by clicking the Start button and searching for "Server Manager".

Once Server Manager is open, click on "Manage" in the top right corner, and then select "Add Roles and Features". This will launch the "Add Roles and Features Wizard". In the wizard, click "Next" on the "Before you begin" page. Then, on the "Select installation type" page, choose "Role-based or feature-based installation" and click "Next".

On the "Server Selection" page, select your server from the server pool and click "Next". On the "Server Roles" page, check the box next to "Remote Desktop Services". A pop-up window will appear asking if you want to add the features that are required for RDS. Click "Add Features" to continue. After selecting RDS, click "Next".

On the "Features" page, you can select any additional features you need, but for a basic setup, you can usually just click "Next". On the "Remote Desktop Services" page, you'll get an overview of what RDS is all about. Click "Next" to move on.

Finally, on the "Role Services" page, select the Role Services you want to install. You will typically want to select at least "Remote Desktop Session Host" and "Remote Desktop Licensing". If you want users to be able to connect to a remote desktop, you must install the ‚ÄúRemote Desktop Session Host‚Äù role service. Select the role services you need and click "Next". On the confirmation page, review your selections and click "Install" to begin the installation. The installation process may take a few minutes. Once the installation is complete, you'll need to configure the RDS services. Once the installation is complete, close the wizard and move on to the next step: configuring the RDS.

Configuring Remote Desktop Services

Alright, now that we've installed the Remote Desktop Services role, let's configure it. After the installation is complete, you might be prompted to reboot your server. Go ahead and do that. Once your server restarts and you log back in, you will see the RDS configuration tasks within Server Manager.

Open Server Manager and navigate to "Remote Desktop Services" in the left-hand menu. You should see a few configuration steps listed there. The first thing you'll want to do is to set up the "RD Session Host". Click on "RD Session Host" and then choose "Configure RD Session Host". Follow the on-screen prompts. This includes specifying the licensing mode (per-user or per-device ‚Äì remember, you'll need to have purchased CALs), and assigning an RD Licensing server.

Next, it‚Äôs crucial to configure RD Licensing. If you've installed the RD Licensing role service, you will now need to activate the license server. Right-click on the server name and select "RD Licensing Manager". Right-click on the server name and select "Activate Server". You'll be guided through the activation process, where you'll need to choose your activation method and provide the necessary information.

After activating the server, you‚Äôll need to install your CALs. In RD Licensing Manager, right-click on the server name, select "Install Licenses", and follow the prompts. You'll be asked to provide your license key (the one you got when you purchased your CALs). The wizard will guide you through the process. After the installation is complete, you can verify that the licenses are installed correctly by checking the license status within the RD Licensing Manager.

Once you've configured these settings, you're well on your way to a functional RDS environment. Remember to test the connection to ensure everything is working correctly.

Activating the Remote Desktop Licensing Server

Now that we've installed the RD Licensing role and the basic settings, let's dive into activating the Remote Desktop Licensing server. This is an essential step to ensure your users can connect with their purchased CALs. Open Server Manager and go to "Remote Desktop Services" in the left-hand menu. There, you'll see the overview of your RDS deployment. If you haven't opened it already, find RD Licensing Manager and open it. You can also search in the start menu for "RD Licensing Manager".

Once RD Licensing Manager is open, you will see the name of your server in the console tree. Right-click on your server name and select "Activate Server". This action will launch the activation wizard. In the activation wizard, you will be prompted to select an activation method. You have three options: automatic, web, or telephone. The automatic activation method is the easiest and most recommended option. This requires an internet connection and will connect with the Microsoft activation servers. The web activation method is used if you have internet access on another computer. The telephone activation method is if you don't have internet access and want to call Microsoft to activate. Follow the prompts for your chosen activation method and provide any required information.

After selecting your preferred activation method, the wizard will guide you through the remaining steps. You will be prompted to enter your company information and other details. After the activation is complete, the license server will be activated, and it's ready for CALs installation. Ensure the activation is successful and note any important information, such as the license server name and status. If you run into any issues during activation, make sure you have a stable internet connection. You can check the Event Viewer in the Windows Server to look for any error messages if needed.

Installing the Remote Desktop Services CALs

Alright, now it's time to install those CALs that you've purchased. Remember those CALs are the keys to unlocking remote access for your users. The activation of the licensing server is completed, the next step is installing the licenses. In RD Licensing Manager, right-click on your server name and select "Install Licenses". This will start the installation wizard. In the wizard, select your license program type. You'll typically choose the program type based on how you purchased your licenses (e.g., Open License, Retail, etc.). Make sure you have your license program information handy.

Next, you'll be prompted to enter the license code. This is the unique code you received when you purchased your CALs. Input the correct license code and follow the on-screen prompts. Make sure that you enter the license code carefully because this is crucial for the installation process. After entering the license code, the wizard will contact Microsoft to validate the code and install the licenses. If the validation is successful, the licenses will be installed on your server.

Once the installation is complete, you can check the status of your installed licenses in the RD Licensing Manager. You should see the number of licenses installed, their type (User or Device), and their status. Verify that the licenses are installed correctly, and they match the number of licenses you purchased. After installing the CALs, you can also set up the licensing scope. If you've configured the licensing scope properly, then the server will be able to allocate the CALs to the users or devices that connect to it.

Once you've installed and verified the CALs, the RDS setup is complete. Users should now be able to connect to your Windows Server 2019 environment remotely. Remember, if you encounter any issues during installation, double-check your license keys and make sure you have internet connectivity if you‚Äôre using online activation. It's also a good idea to review the Event Viewer for any error messages that might provide additional clues.

Verifying the Installation and Troubleshooting

Once you've completed the steps, let's make sure everything is working correctly. The final step is to verify that the installation and configuration is successful. Testing is a very important step of this process, that will ensure the RDS and CALs are functional. First of all, try to connect to your server using a Remote Desktop Connection client. On a Windows computer, you can simply search for ‚ÄúRemote Desktop Connection‚Äù in the Start menu and launch the app. Enter the server's IP address or hostname and try to connect. If you can connect successfully, you're off to a good start!

If you have trouble connecting, double-check your RDS configuration settings in Server Manager. Make sure the RD Session Host role is properly configured. Also, verify that the Remote Desktop is enabled on the server. You can check this by going to System Properties (search for it in the Start menu), and then selecting the "Remote" tab. Ensure that "Allow remote connections to this computer" is selected.

Check the status of your RD Licensing server in the RD Licensing Manager. Confirm that it‚Äôs activated, and the licenses are installed. Also, review the Event Viewer for any error messages related to RDS or licensing. The Event Viewer is a valuable tool for troubleshooting. It records any issues or errors that occur on your server, providing valuable clues as to what might be going wrong. Make sure you're using the correct credentials when trying to connect remotely. If you're using a domain account, make sure the user is a member of the "Remote Desktop Users" group. If you encounter any licensing issues, double-check that you have the correct CAL type (User or Device) for your users or devices. If you've tried all of these troubleshooting steps and you still can't connect, consult the official Microsoft documentation. Good luck!

Conclusion: You Did It!

And there you have it, guys! You've successfully navigated the process of installing and configuring Remote Desktop Services and CALs on your Windows Server 2019. Now, your users can connect remotely and enjoy the benefits of RDS. Remember to always keep your licenses up to date and follow best practices for security. Congrats on setting up RDS. Happy remote accessing!
