Diagnosing and Resolving the Network Connectivity Issue

Alright, let's roll up our sleeves and dive into this network connectivity issue. This is the big kahuna preventing your distributed rendering from working, so let's break it down and troubleshoot like pros. The error message Cannot connect to host 192.168.1.119:8188 ssl:default [The remote computer refused the network connection] is our key clue here. It tells us that a worker node tried to connect to the master node (at IP address 192.168.1.119 on port 8188) but the connection was refused.

Here's a step-by-step approach we can take to diagnose and fix this:

1. Verify the Master Node's IP Address

First things first, let's make sure that 192.168.1.119 is indeed the correct IP address of your master node. It might seem obvious, but it's always good to double-check. You can usually find the IP address by:

  • Windows: Opening the Command Prompt and typing ipconfig.
  • macOS/Linux: Opening the Terminal and typing ifconfig or ip addr.

Look for the IPv4 address associated with your network adapter. If the IP address is different from 192.168.1.119, you'll need to update the configuration on your worker nodes to point to the correct IP.

2. Check ComfyUI's Configuration for Distributed Rendering

Next, let's dive into ComfyUI's configuration files. There might be some settings related to distributed rendering that need tweaking. Look for any configuration files (like a config.json or similar) that might contain settings for the master node's address or port. Ensure that the IP address and port (8188 in this case) are correctly configured.

3. Firewall Inspection (Round Two!)

You mentioned that you've opened the full port range on your firewall, which is a great start! However, firewalls can sometimes be tricky, so let's double-check a few things:

  • Specific Rules: Instead of opening the entire port range, try creating specific rules for port 8188 (or whichever port ComfyUI is using for distributed communication) for both TCP and UDP protocols. This can sometimes be more reliable.
  • Direction: Make sure you have rules allowing both inbound and outbound connections on port 8188 for both the master and worker nodes. The worker needs to connect to the master (outbound), and the master might need to respond (inbound).
  • Windows Firewall Advanced Security: If you're on Windows, the Windows Firewall with Advanced Security is a powerful tool. Make sure you've created rules in both the Inbound Rules and Outbound Rules sections.
  • Other Firewalls: If you have any other firewalls (like a hardware firewall on your router or a software firewall like ZoneAlarm), make sure they're also configured to allow connections on port 8188.
