Finding The Lonely Number: Single Element In Sorted Array

Aug 9, 2025 by ADMIN 58 views

Hey everyone! Let's dive into a cool problem from LeetCode: finding the single element in a sorted array. This one's a classic, and we'll break it down step by step. The goal is to find that one number that only shows up once in an array where all the other numbers appear twice.

The Problem: Spotting the Odd One Out

So, imagine you've got a sorted array filled with numbers. The catch? Every number appears twice, except for one sneaky little number that's all alone. Your mission, should you choose to accept it, is to find that single element. For example, in the array [1,1,2,3,3,4,4,8,8], the answer is 2. Easy peasy, right? Well, there's a twist. You gotta do it efficiently. We're aiming for a solution that runs in logarithmic time, or O(log n), which means we need to think smartly about how we search through the array. No brute-force approaches allowed here, guys! Let's get this sorted! The array is sorted, which is a huge hint that we should be thinking about binary search. Binary search is like a super-powered version of guessing. We start in the middle and then eliminate half the array with each step, zeroing in on the answer. This way is much faster than checking every element one by one. We will go over the problem, the core ideas, and then how the code works. Understanding this problem will give you some new ways to look at arrays and how to use binary search in creative ways.

Let's get into the details. The main idea of the problem is the following: the array is sorted and has a special pattern. The single element separates the duplicate elements, so we can use binary search to find that element. The key is to compare the middle element with its neighbors and eliminate half of the array. For example, if the middle element is at an even index, and the next element is the same, the single element must be in the right half of the array. If the middle element is at an odd index, and the previous element is the same, the single element must be in the right half. This helps narrow down our search space. So, we can keep going until we find the single element. Binary search is such a versatile tool, and we can use it to efficiently solve all sorts of problems. Let's think about the constraints, too. The array will always contain at least one element. The single element will always be present. The other elements will appear exactly twice. Understanding these constraints will help us determine the edge cases. Now, let's explore how to solve it using the binary search method.

Let's get our hands dirty, shall we? We will examine the core concepts, explore the algorithm, and delve into the code implementation. We will make sure that everything is easy to digest. So, how does the algorithm work? We start by setting our left and right pointers to the start and end of the array. We will then calculate the mid index. The most interesting step is checking the mid element against its neighbors. If the middle element is at an even index and the next element is the same, we move our left pointer to mid + 2. If the middle element is at an odd index and the previous element is the same, we move our left pointer to mid + 1. Otherwise, we can move our right pointer to mid - 1. We keep repeating this until left and right meet, at which point we've found the single element. We will get a lot of practice by going through examples. Let's take the array [1,1,2,3,3,4,4,8,8] and apply our algorithm step by step. Initial setup: left = 0, right = 8. mid = 4. nums[mid] == 3, nums[mid - 1] == 3. So, left = mid + 1 = 5. Repeat until the element is found.

Binary Search to the Rescue: The Core Idea

The heart of our solution is binary search. Since the array is sorted, we can use binary search to quickly narrow down the search space. The main idea is this: When you compare the middle element with its neighbors, you can determine which half of the array contains the single element. Here's the trick: If the middle element is at an even index and the next element is the same, the single element must be in the right half. If the middle element is at an odd index and the previous element is the same, the single element is also in the right half. Otherwise, the single element is in the left half. See how we systematically eliminate half of the array with each comparison? That's the magic of binary search! This approach ensures that we can solve the problem in logarithmic time, which is super efficient. Let's break down the process in more detail, step by step. We start with two pointers, left and right, pointing to the start and end of the array. Then, we calculate the middle index. Next, we compare the element at the middle index with its neighbors. Based on the comparison, we adjust the left or right pointer to narrow down the search space. We repeat these steps until the left and right pointers meet. The element at that index is our single element! Binary search, used correctly, is incredibly powerful. It’s a must-know technique for any programmer. The use of binary search lets us solve this problem in logarithmic time complexity.

One of the cool things about binary search is that it's not just about finding the value; it's about finding the position. We use the middle element to tell us whether the target (in this case, the single element) is to the left or to the right. This is how binary search efficiently searches through a sorted dataset, cutting the search area in half each time. By understanding how binary search works, you can apply it to various other problems involving sorted data. It is great at optimization. The beauty of binary search is its efficiency. It makes the algorithm extremely fast, especially for large arrays. Understanding the details can help you solve similar problems in the future. Being able to identify when to use binary search will become a key skill for you.

Cracking the Code: Implementation and Walkthrough

Alright, let's get to the code! I'll walk you through a Python implementation, but the concepts apply to any language. Here's the basic structure:

def single_non_duplicate(nums):
    left, right = 0, len(nums) - 1

    while left < right:
        mid = (left + right) // 2

        # Check if mid is even and the next element is the same
        if mid % 2 == 0 and nums[mid] == nums[mid + 1]:
            left = mid + 2
        # Check if mid is odd and the previous element is the same
        elif mid % 2 != 0 and nums[mid] == nums[mid - 1]:
            left = mid + 1
        else:
            right = mid

    return nums[left]

Let's break it down line by line.

  1. Initialization: We set left and right pointers to the start and end of the array. This defines our search space.
  2. Binary Search Loop: The while left < right loop is the heart of the algorithm. It continues as long as our search space isn't narrowed down to a single element.
  3. Midpoint Calculation: mid = (left + right) // 2 calculates the middle index. We use integer division (//) to ensure that mid is an integer.
  4. Comparison: This is where the magic happens! We check if the element at mid has a duplicate. The key here is checking the parity of mid. If the middle index is even and its next element is the same, we know the single element must be in the right half. Likewise, if the middle index is odd and its previous element is the same, then the single element must be in the right half. This is because the duplicates always come in pairs.
  5. Adjusting Pointers: Based on the comparison, we adjust the left or right pointer to narrow down the search space.
  6. Return: When the loop finishes, left and right will point to the same index, which holds the single element. We return nums[left]. This is the element we've been searching for.

  • The loop continues with left = 6, right = 8. mid = 7, nums[7] == nums[8] (8 == 8). The index is odd. We shift the left to mid + 1 = 8.
  • The loop terminates since left is not less than right. The single element is nums[left], or nums[8] == 8. However, after the first iteration, the value of mid will be 4, then since nums[4] and nums[3] are the same, then we move left to 6. In the second iteration, since the value of mid is 7, which is odd and nums[7] == nums[8], then we move left to 8. The algorithm will then terminate and return nums[8] which is 8.
