The Solution: A Bash Script to the Rescue

The key to saving all tabs in all Geany windows lies in a bit of scripting magic. We'll use a Bash script combined with command-line tools to achieve this. Here‚Äôs the breakdown of the script:

#!/bin/bash

# Get a list of Geany window IDs
window_ids=$(wmctrl -l | grep "Geany" | awk '{print $1}')

# Loop through each window ID
for window_id in $window_ids; do
  # Get the window title
  window_title=$(wmctrl -l -p -i -r "$window_id" | awk '{$1=$2=$3=$4=$5=""; print $0}')
  # Create a filename based on the window title
  filename="~/geany_session_$(echo "$window_title" | tr -dc A-Za-z0-9_)".txt
  
  # Save the list of documents for the window
  geany --list-documents > "$filename"
  echo "Saved tabs from window '$window_title' to $filename"
done

echo "All Geany tabs saved!"

Let‚Äôs walk through this script step by step to understand how it works and why it‚Äôs so effective.

First, the script starts with the shebang #!/bin/bash, which tells the system to use Bash to execute the script. This is a standard practice for Bash scripts and ensures that the script is run with the correct interpreter.

Next, the script uses the wmctrl command to get a list of all window IDs. wmctrl is a command-line tool that interacts with the window manager, allowing you to perform various window operations. The -l option lists all open windows, and the grep "Geany" part filters the list to only include windows with ‚ÄúGeany‚Äù in their title. The awk '{print $1}' then extracts the first field, which is the window ID.

The script then enters a for loop that iterates through each window ID found. For each window, it retrieves the window title using wmctrl -l -p -i -r "$window_id". The -l -p -i -r options specify that we want to list window information, including the process ID, window ID, and window title. The awk '{$1=$2=$3=$4=$5=""; print $0}' part removes the first five fields (which are not part of the title) and prints the rest, giving us the window title.

Now, the script creates a filename based on the window title. This is done to ensure that each saved session has a unique filename. The filename variable is constructed as ~/geany_session_$(echo "$window_title" | tr -dc A-Za-z0-9_)_.txt. This means the filename will start with ‚Äú~/geany_session_‚Äù, followed by the window title (with any non-alphanumeric characters removed), and end with ‚Äú.txt‚Äù. The tr -dc A-Za-z0-9_ command is used to remove any characters that are not letters, numbers, or underscores, ensuring a valid filename.

Inside the loop, the script uses the geany --list-documents command to save the list of open tabs for the current window. The output is redirected to the filename we just created using > "$filename". This effectively saves the list of open files for that window to a text file.

The script then prints a message to the console, indicating that the tabs from the current window have been saved and the filename where they were saved. This provides feedback to the user, letting them know that the script is working and what it‚Äôs doing.

Finally, after the loop has finished processing all Geany windows, the script prints a message ‚ÄúAll Geany tabs saved!‚Äù to the console, confirming that the entire process is complete.

This script is a powerful tool for anyone who uses Geany extensively, especially across multiple windows. It automates the process of saving your work, ensuring that you never lose your progress. Let's see how you can use it.

How to Use the Script

  1. Save the Script: Copy the script above and save it to a file, for example, save_geany_tabs.sh.
  2. Make it Executable: Open your terminal and navigate to the directory where you saved the script. Then, run the command chmod +x save_geany_tabs.sh to make the script executable. This step is crucial because it tells the operating system that the file is an executable program.
  3. Run the Script: Simply run the script from your terminal using ./save_geany_tabs.sh. The script will then go through each Geany window and save the list of open tabs to a separate file in your home directory.

Now, you'll find a bunch of .txt files in your home directory, each containing the list of open files for a specific Geany window. The filenames are based on the window titles, making it easy to identify which file corresponds to which window.

Restoring the Tabs

Okay, so you've saved your tabs. Awesome! But how do you restore them? Here's another handy script for that:

#!/bin/bash

# Check if a filename is provided as an argument
if [ -z "$1" ]; then
  echo "Please provide a filename as an argument."
  exit 1
fi

filename="$1"

# Check if the file exists
if [ ! -f "$filename" ]; then
  echo "File '$filename' not found."
  exit 1
fi

# Read the list of files from the specified file
while IFS= read -r file; do
  # Open each file in Geany
  geany "$file"
done < "$filename"

echo "Restored tabs from $filename"

Let's break down this restoration script. It‚Äôs designed to read a list of files from a specified text file and open them in Geany. This script is a perfect complement to the saving script we discussed earlier, allowing you to quickly restore your saved sessions.

First, similar to the saving script, this script starts with the shebang #!/bin/bash, ensuring that it's executed with Bash.

The script begins with an important check: it verifies if a filename is provided as an argument. This is done using the conditional statement if [ -z "$1" ]; then. Here, $1 represents the first argument passed to the script, which should be the filename containing the list of tabs to restore. If $1 is empty (meaning no filename was provided), the script prints an error message ‚ÄúPlease provide a filename as an argument.‚Äù and exits with a status of 1, indicating an error.

Next, the script assigns the provided filename to the variable filename. This is a simple step but makes the script more readable and easier to maintain.

After that, the script checks if the specified file exists using if [ ! -f "$filename" ]; then. The -f option tests if the file exists and is a regular file (not a directory or other special file). If the file does not exist, the script prints an error message ‚ÄúFile ‚Äò$filename‚Äô not found.‚Äù and exits with a status of 1.

The core of the script is the while loop, which reads the list of files from the specified file. The loop uses the IFS= read -r file command, which reads each line from the file and assigns it to the variable file. The IFS= part ensures that leading and trailing whitespace is preserved, and the -r option prevents backslash escapes from being interpreted.

Inside the loop, the script executes the command geany "$file". This command opens the file specified by the $file variable in Geany. As the loop iterates through each line in the file, each file listed is opened in Geany.

Finally, after the loop has processed all the files in the list, the script prints a message ‚ÄúRestored tabs from $filename‚Äù to the console, confirming that the restoration process is complete.

This script is an essential part of the workflow for saving and restoring Geany sessions. It allows you to quickly reopen all the files you were working on, making it easy to switch between projects or pick up where you left off. Now, let's see how you can use this script in practice.

How to Use the Restore Script

  1. Save the Script: Save this script as, for example, restore_geany_tabs.sh.
  2. Make it Executable: Run chmod +x restore_geany_tabs.sh in your terminal.
  3. Run the Script: To restore a session, run ./restore_geany_tabs.sh ~/geany_session_YourWindowTitle.txt, replacing ~/geany_session_YourWindowTitle.txt with the actual filename of the session you want to restore.

Geany will then open all the files listed in the specified session file. How cool is that?

Automating the Process

To make things even smoother, you can automate the saving process. Consider adding a cron job or a keyboard shortcut to run the save_geany_tabs.sh script regularly. This way, you can ensure that your sessions are always backed up without having to manually run the script each time.
