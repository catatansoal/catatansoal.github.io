Project Overview: The Need for Ephemeral File Sharing

In today‚Äôs digital age, sharing files is a common task, but what about those files that contain sensitive information? You need a way to ensure that these files are only accessed by the intended recipient and that they don't linger around on servers, posing a security risk. That's where a One-Time-View File Server comes into play. This system is designed to be a secure, temporary file-sharing solution that allows for the transfer of confidential files via a single-use link.

Once the file is accessed‚Äîwhether it's downloaded or viewed‚Äîor after a user-defined expiry time, the file is automatically deleted from the server. This ensures ephemeral and private transmission, making it an ideal solution for sending sensitive documents, credentials, licenses, and more over a Local Area Network (LAN) or the Internet. The system can also generate QR codes, making sharing across devices incredibly seamless.

The core idea behind this project is to provide a secure method for transferring sensitive data without the risk of long-term storage or unauthorized access. By employing a one-time-view mechanism, the server ensures that once a file is accessed, it is immediately and permanently removed. This approach significantly reduces the attack surface and enhances overall security. Think about it ‚Äì no file sitting on a server means no file to be potentially compromised in a data breach. This is particularly crucial in scenarios where files contain personal identifiable information (PII), financial data, or proprietary business secrets.

One of the critical aspects of this system is its adaptability to different network environments. Whether you are on a LAN or need to share files over the Internet, this server can be configured to meet your needs. For LAN use, the setup is straightforward, requiring minimal configuration. However, sharing files over the Internet adds another layer of complexity, primarily due to the need for a publicly accessible endpoint. QR code generation is another feature that adds a layer of convenience and security. This is especially useful for transferring files between devices, such as from a computer to a mobile phone, without the hassle of emailing or using cloud storage services.

Key Features: What Makes This Server Special?

Let‚Äôs break down the main features that make this One-Time-View File Server a powerhouse of secure file sharing:

Self-Destructing File Links

The heart of this system is its ability to create file links that self-destruct. Files are deleted either after one successful download or after a configurable timeout period. This ensures that your sensitive data isn't left lingering on the server.

The self-destructing nature of the file links is crucial for maintaining the confidentiality of the shared data. Imagine you need to send a document containing confidential client information. By using a one-time-view file server, you can rest assured that once the recipient has accessed the file, it is automatically deleted, reducing the risk of unauthorized access in the future. This feature is particularly valuable in industries such as finance, healthcare, and legal, where data protection is paramount.

The flexibility of configuring the deletion criteria‚Äîeither after one download or after a specified time‚Äîadds an extra layer of control. For instance, you might set a timeout period for highly sensitive documents that you want to ensure are not accessible beyond a certain timeframe, even if they haven't been downloaded. Conversely, for less sensitive files, you might rely solely on the one-time download criterion. The configurable timeout is also useful in scenarios where the recipient might not be able to download the file immediately but needs assurance that it will remain available for a short period.

One-Time Access Token

Each upload generates a secure, random URL token. These links are cryptographically hard to guess, adding an extra layer of security to prevent unauthorized access. This one-time access token mechanism is a foundational element in the overall security architecture of the server. When a file is uploaded, the server generates a unique, random URL token that serves as the access key for that specific file. This token is not only random but also cryptographically strong, making it extremely difficult for anyone to guess or brute-force the URL.

The strength of the token lies in its complexity and unpredictability. By using robust cryptographic algorithms to generate these tokens, the server ensures that the likelihood of someone stumbling upon a valid URL is infinitesimally small. This is in stark contrast to more simplistic URL schemes that might use sequential IDs or predictable patterns, which are inherently vulnerable to unauthorized access attempts.

The use of a secure, random URL token also facilitates the implementation of the one-time-view functionality. The server tracks whether the token has been used and, upon the first access, either deletes the file immediately or sets it for deletion after the specified timeout. This process ensures that the shared file is accessible only via the unique token and only for a limited time, providing a strong defense against unauthorized viewing or downloading.

Works Over LAN/Internet

Whether you‚Äôre sharing files on a Local Area Network (LAN) or over the internet, this server has got you covered. It requires zero configuration for LAN use and offers optional ngrok or dynamic DNS integration for external access. The versatility of this feature is one of the key advantages of the One-Time-View File Server, making it a valuable tool for a wide range of users and scenarios. The server is designed to operate seamlessly within a LAN environment, requiring minimal setup and configuration. This makes it an excellent solution for sharing files within a home network, an office, or any other local network infrastructure.

For LAN use, the server can be deployed quickly and easily, often without needing to make any changes to the network‚Äôs existing setup. Users can simply access the server through its local IP address, and the one-time-view functionality ensures that shared files remain secure even within the network. This is particularly useful for transferring files between computers on the same network, such as sharing documents, images, or videos with colleagues or family members.

However, the true power of this server lies in its ability to facilitate secure file sharing over the internet. Sharing files over the internet introduces additional complexities, primarily due to the need for a publicly accessible endpoint. This is where technologies like ngrok or dynamic DNS integration become invaluable. ngrok is a tool that creates secure tunnels to localhost, allowing you to expose your local server to the internet without needing to configure port forwarding or deal with firewall settings. This makes it incredibly easy to share files with anyone, anywhere, as long as they have the unique URL generated by the server.

Dynamic DNS, on the other hand, is a service that maps a static hostname to a dynamic IP address. This is particularly useful for users who have a dynamic IP address assigned by their internet service provider (ISP), as it allows them to access the server using a consistent hostname even if their IP address changes. The combination of the One-Time-View File Server with ngrok or dynamic DNS integration provides a robust solution for secure file sharing over the internet, ensuring that sensitive data can be transferred privately and efficiently.

Want to share files from your computer to your phone? No problem! The server can auto-generate QR codes for file URLs, making mobile-to-PC transfers a breeze. The inclusion of QR code support is a clever addition that enhances the usability and convenience of the One-Time-View File Server. QR codes provide a quick and efficient way to access URLs, especially when transferring files between devices. Instead of having to type a long and complex URL, users can simply scan the QR code with their smartphone or tablet, instantly accessing the file.

This feature is particularly useful for scenarios where you need to transfer files from your computer to your mobile device, or vice versa. For example, imagine you have a document on your computer that you want to view on your smartphone. With QR code support, you can simply upload the file to the One-Time-View File Server, generate the QR code, and scan it with your phone. The file will then be accessible on your mobile device, and after you‚Äôve viewed or downloaded it, it will be automatically deleted, ensuring your data remains secure.

The generation of QR codes is typically handled by a library or module within the server's software stack. The qrcode Python library, or a JavaScript QR renderer, can be used to create these codes dynamically. This allows the server to generate a QR code each time a file is uploaded, ensuring that every file has its own unique code that corresponds to its one-time-view URL.

The convenience of QR code support extends beyond personal use cases. In a business setting, for example, it can be used to quickly share presentations or documents with colleagues during a meeting. Minimal Logs / No Tracking

Privacy is a priority. The server stores minimal logs and doesn't track user data. Only access time and IP are logged for single-use validation. The commitment to minimal logs and no tracking is a testament to the One-Time-View File Server‚Äôs focus on privacy and security. In an age where data privacy is a growing concern, this feature sets the server apart by ensuring that user data is not stored or tracked beyond what is absolutely necessary for the server to function.

The server's design intentionally avoids storing personal information or tracking user activity. The only logs that are maintained are those essential for validating the single-use nature of the file links. Specifically, the server logs the access time and the IP address of the user who accessed the file. This information is used to ensure that each file is accessed only once, as intended. Once the file has been accessed and the one-time-view condition has been met, this minimal log data is also typically purged, further enhancing privacy.

The decision to limit logging and avoid tracking is a conscious choice to reduce the risk of data breaches and protect user privacy. The tech stack for the One-Time-View File Server is a blend of modern and robust technologies, carefully selected to ensure performance, security, and ease of deployment. The backend, which handles the core logic of the server, is primarily built using Python, with frameworks like FastAPI or Flask being the top choices. FastAPI is a modern, high-performance web framework for building APIs with Python. Flask, on the other hand, is a lightweight and flexible web framework that provides the essentials for building web applications without imposing strict architectural constraints. The frontend, which is responsible for the user interface, is built using HTML and TailwindCSS. Temporary storage is a critical component of the One-Time-View File Server, as it determines how files are stored and managed before they are deleted. The server can use either in-memory storage (RAM) or a tempfs file system for temporary storage. Storing files in RAM offers the fastest performance but is limited by the available memory. tempfs, which is a temporary file system stored in memory or swap space, provides a good balance between performance and storage capacity. For environments with limited resources, the server can also use disk storage, but this option requires a cleanup mechanism to ensure that files are deleted promptly.

QR code generation is handled by either the qrcode Python library or a JavaScript QR renderer. The qrcode library is a popular Python library for generating QR codes, while JavaScript QR renderers allow for client-side QR code generation, reducing the load on the server.

Security is a paramount concern, and the One-Time-View File Server employs several security measures to protect user data. HTTPS can be used to encrypt communication between the client and the server, ensuring that data is transmitted securely. SHA256 tokens are used to generate secure, random URLs for accessing files, making it difficult for unauthorized users to guess or brute-force the URLs.

Deployment options for the server are flexible, allowing it to be deployed in a variety of environments. For LAN use, the server can be deployed directly on a local network without any special configuration. For internet access, the server can be deployed using ngrok, IPv6, or a reverse proxy like Nginx. ngrok provides a simple way to expose the server to the internet, while IPv6 offers a global addressing scheme that eliminates the need for Network Address Translation (NAT). Nginx, a popular web server and reverse proxy, can be used to securely expose the server to the internet and handle load balancing and caching.

Inspiration: Where Did This Idea Come From?

This project draws inspiration from services like Firefox Send (discontinued), Snapdrop, and WeTransfer. However, it focuses on privacy, ephemeral access, and ease-of-use in LAN or small office setups.

The One-Time-View File Server is inspired by a variety of existing services and concepts that prioritize secure and temporary file sharing. Services like Firefox Send (which has unfortunately been discontinued), Snapdrop, and WeTransfer have demonstrated the value and demand for tools that allow users to share files securely and privately. These services have pioneered various approaches to ephemeral file sharing, such as self-destructing links, encryption, and temporary storage. Firefox Send, in particular, was a popular service that allowed users to share files with end-to-end encryption and self-destructing links, ensuring that the files were only accessible to the intended recipient and were automatically deleted after a specified time or number of downloads. Its focus on privacy and security made it a trusted tool for sharing sensitive information.

Snapdrop is another service that has inspired the One-Time-View File Server. Snapdrop is a local file-sharing application that uses peer-to-peer (P2P) technology to transfer files directly between devices on the same network. It eliminates the need for a central server, providing a fast and private way to share files. Its simplicity and ease of use have made it a popular choice for sharing files within a local network. WeTransfer is a well-known file-sharing service that allows users to send large files to others via email. While WeTransfer does not offer self-destructing links or end-to-end encryption by default, it does provide a convenient way to share large files with recipients who may not have access to cloud storage services. Its popularity and widespread use have made it a standard for file sharing in many industries.

The One-Time-View File Server builds upon the ideas and concepts pioneered by these services, but it also aims to address some of their limitations and focus on specific use cases. In particular, the One-Time-View File Server emphasizes privacy, ephemeral access, and ease of use in LAN or small office setups. While services like WeTransfer are designed for general file sharing over the internet, the One-Time-View File Server is optimized for secure and private file sharing within a local network or small office environment. This focus allows it to offer features and capabilities that are specifically tailored to these use cases, such as LAN-friendly configuration, QR code support for device-to-device transfers, and minimal logging and tracking to protect user privacy.
