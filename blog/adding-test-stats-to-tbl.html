The Core Problem: Missing Test Statistics in tbl_summary

So, the deal is, when you use tbl_summary along with add_p(), you get a sweet table with p-values, which is awesome. But, often, you're left wondering, "What exactly was the test statistic for that p-value?" Was it a t-statistic, a chi-square, or something else? This is where things get interesting. The initial output from tbl_summary and add_p() doesn't directly give you the test statistic. But, don't worry, there is a solution! The goal is to tweak the output to display not only the p-value, but also the associated test statistic, providing a much richer context for your results. Adding the test statistic provides the full story. This is crucial for both clarity and making sure your work is reproducible. It helps you keep track of the underlying data.

Here's a quick example using the basic tbl_summary format. Consider a situation where you're comparing the ages and grades of patients based on some treatment (trt): This helps you to fully understand and be aware of all the test results.

library(gtsummary)

Solution: Customizing add_p() with Test Statistics

Okay, time to roll up our sleeves and get our hands dirty with the solution. The key lies in customizing the add_p() function. We'll use it, but in a way that forces it to show the test statistic alongside the p-value. This can sometimes be a bit tricky, as the exact method depends on the tests you're running. It's about presenting the full picture, not just a fragment. Remember, we're making sure your results are as clear as possible. Hereâ€™s a modified approach you can start with, which involves extracting the relevant statistic from the test results and including it in your table. It requires a bit more coding, but it pays off big time in terms of clarity. Let's make it happen, and make sure your tables stand out.

# Assuming 'trial' is your data frame
trial |>
  tbl_summary(by = trt, include = c(age, grade)) |>
  add_p(test = list(all_continuous() ~ "t.test",
                  all_categorical() ~ "chisq.test")) |>
  modify_header(update = list(p.value ~ "P-value", statistic ~ "Test Statistic")) |>
  modify_fmt_fun(statistic ~ function(x) style_sigfig(x, digits = 2))

In this code: The test argument in add_p() helps specify the tests. The modify_header function lets you rename the columns. The modify_fmt_fun function formats the test statistics. This is the essential bit, right here. Remember to modify the tests to be compatible with your variables.

This example is a solid starting point. However, youâ€™ll likely need to adapt it based on the type of tests you're running (t-tests, chi-squared, etc.). Each test will have its own way of producing the statistic, and we'll need to extract it appropriately. The modify_header and modify_fmt_fun functions are super important as they provide the flexibility to customize the output. Let's not forget the aesthetic part of your table. This customization helps make the output visually appealing.

Detailed Example with a T-Test

Let's illustrate this with a detailed example, focusing on a t-test. This will give you a tangible example of how to apply the steps we've outlined. I'll show you how to get the t-statistic into your summary table. We'll work with some sample data to make things super clear. Then, you can use this as a template. So, here's a specific example. This will give you a hands-on understanding.

library(gtsummary)
library(dplyr)

# Sample data (replace with your data)
trial_data <- trial |>
  select(trt, age) |>
  na.omit()

# Perform t-test
t_test_result <- t.test(age ~ trt, data = trial_data)

# Create tbl_summary with custom p-value and test statistic
trial_data |>
  tbl_summary(by = trt, include = age) |>
  add_p(test = list(all_continuous() ~ "t.test")) |>
  modify_header(update = list(p.value ~ "P-value",
                             statistic ~ paste0("T-statistic: ", round(t_test_result$statistic, 2))))

In this example, we first perform a t-test to compare the age variable between treatment groups. We then integrate the t-statistic (t_test_result$statistic) into the table. The code includes sample data, the t-test itself, and integration into tbl_summary. This way, you can easily see the t-statistic. The modify_header function is used to add the t-statistic. Make sure the sample data matches what you're working with. This example lets you extract and show the test statistic. This helps you to better describe your analysis.
