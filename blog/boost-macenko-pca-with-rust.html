Boost Macenko PCA With Rust: Faster Color Normalization

Aug 1, 2025 by ADMIN 56 views

Hey guys! I'm super excited to share my journey into optimizing the Macenko PCA implementation using Rust. For those who don't know, PCA (Principal Component Analysis) is a powerful technique used in color normalization, especially in digital pathology and image analysis. This project was born out of my desire to dive deeper into Rust, and what better way to learn than by tackling a real-world problem?

The Challenge: Speeding Up Macenko PCA

So, I decided to take on the challenge of rewriting the Macenko PCA implementation in Rust. The original implementation is fantastic, but I saw an opportunity to make it even faster. It took a lot of trial and error, but I managed to achieve some impressive speed improvements. It's still a work in progress, especially in terms of memory usage, but I'm thrilled with the progress so far.

Why Rust?

Rust is a systems programming language known for its speed, safety, and concurrency. It's a perfect fit for computationally intensive tasks like PCA. Plus, it was a great excuse for me to get my hands dirty with a new language.

The Idea: An Optional Rust Implementation

My vision is to integrate this Rust implementation as an optional component. The Python version would remain the default, ensuring stability and backward compatibility. But, for those who need that extra speed boost, the Rust version could be enabled as an "if installed" option. This approach allows for flexibility and caters to different user needs. By building my wheel(s) and publishing it on PyPI, you can list it as an optional dependency, and use it if possible. If for some reason it needs maintaining and I drop off the earth, you can just drop that option.

Performance Benchmarks: Rust vs. Python

Now, let's talk numbers! I ran some benchmarks on my Macbook Pro M1 Max (64GB RAM) to compare the performance of the Rust and Python implementations.

Test Image: shape:(24600, 22498, 3) dtype:float64 min:1.0 max:256.0
HTK PCA took 116.65962195396423 seconds
Rust PCA took 36.90314483642578 seconds

The results speak for themselves! The Rust implementation is over 3x faster than the original Python version. To be honest, my first attempt was 3x slower, lol. About 1/3 of the time is spent in OpenBLAS, which is a testament to its efficiency.

Diving Deeper into the Numbers

The Road Ahead: Optimizations and Considerations

My journey doesn't end here. I'm still exploring ways to optimize the Rust implementation further. Memory usage is a key area I'm focusing on. Given that this is my first real Rust project, I know there's plenty of room for improvement.

Float32: A Trade-off Between Precision and Performance?

I'm also considering a float32 implementation. While it might sacrifice some precision compared to float64, it could offer significant performance gains and reduce memory footprint. This could be a valuable option for applications where speed is critical, and slight inaccuracies are acceptable. Considering how much less precise uint8 is, there might be an argument for a float32 implementation, very willing to play with that if you decide to move forward with this and are interested in a faster, lighter, and less precise (probably optional?) float32 version.

Contribution and Collaboration

This brings me to my main question: Would you guys be interested in incorporating this Rust implementation as a contribution? I believe it could be a valuable addition, offering a faster alternative for Macenko PCA. I'm envisioning it as an optional install, with the Python implementation remaining the default. This way, users can choose the version that best suits their needs.

Conclusion: A Faster Future for Color Normalization

I'm incredibly excited about the potential of this Rust implementation. It's been a challenging but rewarding project, and I'm eager to continue pushing the boundaries of performance in color normalization. I'm open to feedback, suggestions, and collaboration. Let's work together to make this even better!

In summary, this Rust implementation of Macenko PCA offers a significant speed improvement over the Python version. While there are still areas to explore, such as memory optimization and float32 implementation, the initial results are promising. The potential for an optional Rust-based Macenko PCA in DigitalSlideArchive and HistomicsTK could provide a valuable performance boost for users.

The key benefits of this Rust implementation include its speed, which is over 3x faster than the Python version, and its potential for further optimization. By offering it as an optional component, users can choose the implementation that best suits their needs, balancing performance and stability.

This project highlights the power of Rust in tackling computationally intensive tasks. Its speed, safety, and concurrency features make it an ideal language for optimizing algorithms like Macenko PCA. The visual comparison of the color normalization results demonstrates the practical impact of this optimization, paving the way for faster and more efficient digital pathology workflows.
