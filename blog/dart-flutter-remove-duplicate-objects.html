Dart/Flutter: Remove Duplicate Objects By Property

Aug 6, 2025 by ADMIN 51 views

Hey guys! Ever find yourself wrestling with duplicate objects in your Dart or Flutter lists? Especially when you're dealing with streams that keep spitting out the same data over and over? It's a common problem, particularly when you're working with sensor data, device updates, or any kind of real-time information. Imagine you're building a fitness app that tracks a user's heart rate. The heart rate sensor might send the same reading multiple times in a row, and you don't want to clutter your app's data with these duplicates. Or, think about an app that monitors the status of smart home devices. If a device's status hasn't changed, you don't need to record the same status update repeatedly. This article will guide you through several effective strategies to remove these pesky duplicates, focusing on identifying objects by a unique property. We'll cover different approaches, from using Set to more complex scenarios where you need to compare objects based on a specific attribute. So, buckle up and let's dive into the world of duplicate removal in Dart and Flutter!

The Problem: Duplicate Objects in Streams

Let's break down the scenario: You're capturing events (in your case, Device objects) from a stream. This stream operates on a timer, meaning it repeatedly picks up the same device and adds it to the stream. This is super common in IoT applications, sensor data processing, or even when polling APIs. The challenge here is that you end up with a list or stream containing multiple Device objects that are essentially the same, based on some unique identifier (like a device ID). You want to clean this data so that you only have one entry per unique device. For instance, if you're building a smart home app, you might be getting repeated status updates from a light bulb. You only need to process the update if the status has actually changed, not every time the stream sends the same information. The goal is to efficiently filter this stream or list, ensuring that each Device object in your collection is unique based on a specific property. This not only saves memory but also prevents unnecessary processing and UI updates, making your app more responsive and efficient. We need a way to look at each Device object, check if we've already seen one with the same ID, and only add it to our list if it's truly a new, unique device.

Solutions for Removing Duplicates

Okay, let's get into the nitty-gritty of how to actually remove those duplicate objects! We'll explore a few different methods, each with its own pros and cons. This way, you can choose the one that best fits your specific needs and coding style.

1. Using a Set for Uniqueness

The simplest and often most efficient way to remove duplicates is by leveraging the power of Set in Dart. A Set is a collection that only allows unique elements. However, by default, Dart determines uniqueness by checking if two objects are the exact same instance in memory (i.e., identical(obj1, obj2) returns true). This won't work if you have two Device objects with the same properties but are different instances. To solve this, you need to override the hashCode and == operators in your Device class. Let's see how this looks in practice:

class Device {
  final String id;
  final String name;

  Device({required this.id, required this.name});

  @override
  bool operator ==(Object other) =>
      identical(this, other) ||
      other is Device && runtimeType == other.runtimeType && id == other.id;

  @override
  int get hashCode => id.hashCode;
}

void main() {
  final devices = [
    Device(id: '1', name: 'Light Bulb'),
    Device(id: '2', name: 'Smart Thermostat'),
    Device(id: '1', name: 'Light Bulb'), // Duplicate
  ];

  final uniqueDevices = devices.toSet().toList();

  uniqueDevices.forEach((device) => print('ID: ${device.id}, Name: ${device.name}'));
  // Expected Output:
  // ID: 1, Name: Light Bulb
  // ID: 2, Name: Smart Thermostat
}

Explanation:

  • hashCode and == Override: We've overridden these methods in the Device class. The == operator now checks if two Device objects have the same id. The hashCode is based on the id as well. This is crucial because the Set uses these methods to determine uniqueness.
  • toSet(): We convert the list of Device objects to a Set. This automatically removes any duplicates based on our overridden == and hashCode methods.
  • toList(): We convert the Set back to a List if you need the result as a list.

Pros:

  • Simple and Concise: This is a very clean and easy-to-read solution.
  • Efficient: Set lookups are generally very fast (O(1) on average).

Cons:

  • Requires Modifying the Class: You need to have control over the Device class to override hashCode and ==. If you don't, you'll need a different approach.
  • Order Not Guaranteed: Set doesn't guarantee the order of elements. If you need to preserve the original order, this might not be the best option.

2. Using where with a Set for Streams

If you're working directly with a stream and want to filter out duplicates as they arrive, you can use the where method in combination with a Set. This approach allows you to process each Device object from the stream and only pass it along if it's the first time you've seen its ID. Let's see how this works:

import 'dart:async';

class Device {
  final String id;
  final String name;

  Device({required this.id, required this.name});
}

void main() async {
  final deviceStream = Stream.fromIterable([
    Device(id: '1', name: 'Light Bulb'),
    Device(id: '2', name: 'Smart Thermostat'),
    Device(id: '1', name: 'Light Bulb'), // Duplicate
    Device(id: '3', name: 'Smart Lock'),
    Device(id: '2', name: 'Smart Thermostat'), // Duplicate
  ]);

  final seenIds = <String>{};

  final uniqueDeviceStream = deviceStream.where((device) => seenIds.add(device.id));

  await for (final device in uniqueDeviceStream) {
    print('ID: ${device.id}, Name: ${device.name}');
  }

  // Expected Output:
  // ID: 1, Name: Light Bulb
  // ID: 2, Name: Smart Thermostat
  // ID: 3, Name: Smart Lock
}

Explanation:

  • seenIds: We create an empty Set called seenIds to keep track of the device IDs we've already encountered.
  • deviceStream.where(...): The where method filters the stream. For each Device object in the stream, the provided function is called.
  • seenIds.add(device.id): This is the key part. The add method of a Set returns true if the element was successfully added (i.e., it wasn't already in the set) and false otherwise. So, the where method only allows Device objects to pass through if their ID is not already in seenIds.

Pros:

  • Works Directly with Streams: This is ideal when you're processing data from a stream and want to filter duplicates on the fly.
  • Doesn't Require Modifying the Class: You don't need to override hashCode and == in the Device class.
  • Preserves Order: The order of elements in the stream is preserved.

Cons:

  • Side Effect: The where method has a side effect (modifying the seenIds set). While this is intentional, it's something to be aware of.
  • Memory Usage: The seenIds set will grow as it stores the IDs of all unique devices. If you're dealing with a very large number of unique devices, this could become a concern.

3. Using a Loop and a List

If you can't modify the Device class and you're not working with a stream, you can use a more traditional loop-based approach. This involves iterating through the list and manually checking for duplicates. While it's not as elegant as the Set-based solutions, it's a reliable option when you have limited control over the data structure.

class Device {
  final String id;
  final String name;

  Device({required this.id, required this.name});
}

void main() {
  final devices = [
    Device(id: '1', name: 'Light Bulb'),
    Device(id: '2', name: 'Smart Thermostat'),
    Device(id: '1', name: 'Light Bulb'), // Duplicate
  ];

  final uniqueDevices = <Device>[];

  for (final device in devices) {
    if (!uniqueDevices.any((d) => d.id == device.id)) {
      uniqueDevices.add(device);
    }
  }

  uniqueDevices.forEach((device) => print('ID: ${device.id}, Name: ${device.name}'));

  // Expected Output:
  // ID: 1, Name: Light Bulb
  // ID: 2, Name: Smart Thermostat
}

Explanation:

  • uniqueDevices: We create an empty list called uniqueDevices to store the unique Device objects.
  • Loop: We iterate through the original devices list.
  • uniqueDevices.any(...): For each Device object in the original list, we check if there's already a Device object in the uniqueDevices list with the same id. If there isn't, we add the current Device object to the uniqueDevices list.

Pros:

  • Doesn't Require Modifying the Class: You don't need to override hashCode and == in the Device class.
  • Preserves Order: The order of elements is preserved.
  • Simple Logic: The logic is straightforward and easy to understand.

Cons:

  • Less Efficient: This is the least efficient of the three methods, especially for large lists. The any method iterates through the uniqueDevices list for each element in the original list, resulting in a time complexity of O(n^2).
  • More Verbose: This approach requires more code than the Set-based solutions.

Choosing the Right Approach

So, which method should you choose? Here's a quick guide:

  • If you can modify the Device class and performance is critical: Use the Set approach with overridden hashCode and ==.
  • If you're working with a stream and want to filter duplicates on the fly: Use the where method with a Set.
  • If you can't modify the Device class and you're not working with a stream: Use the loop-based approach, but be mindful of performance for large lists.

Remember to consider the specific requirements of your project and choose the method that best balances performance, code clarity, and maintainability. Good luck removing those duplicates!
