Ubuntu 20.04 On OVH Bare Metal BYOL Installation Guide

Jul 13, 2025 by ADMIN 55 views

Hey everyone! Are you looking to get Ubuntu 20.04 up and running on your OVH bare metal server using the Bring Your Own Linux (BYOL) method? You're in the right place! I understand it can be a bit tricky, especially when you hit roadblocks like missing scripts or unclear documentation. I've been there, and I'm here to help you navigate the process smoothly. This guide will walk you through a step-by-step process, addressing common issues and providing solutions to ensure a successful installation.

Understanding the BYOL Method

Before we dive into the nitty-gritty, let's quickly clarify what the Bring Your Own Linux (BYOL) method actually means. Essentially, BYOL gives you the freedom to install any operating system you want on your OVH bare metal server. This is super useful if you have specific OS requirements or configurations that aren't offered by OVH's standard installation options. It allows for complete customization, making it ideal for advanced users who need a particular setup.

The power of BYOL comes with the responsibility of managing the OS installation yourself. You're in charge of preparing the disk image, ensuring it's bootable, and configuring the network settings. This might sound intimidating, but don't worry! This guide is designed to break down each step into manageable chunks, making the whole process a lot less daunting. We'll be focusing on Ubuntu 20.04 in this guide, but many of the principles can be applied to other Linux distributions as well.

Common Challenges with BYOL Installations

One of the most common hurdles when using BYOL is dealing with the initial image preparation. You need to create a disk image that's not only a functional Ubuntu installation but also compatible with OVH's infrastructure, especially the network and bootloader. This often involves setting up the correct partitioning scheme, installing the necessary bootloader, and configuring the network interfaces. If these steps aren't done correctly, the server might fail to boot, or you might encounter network connectivity issues.

Another challenge is the documentation itself. Sometimes, the official documentation might be outdated or incomplete, leaving you scratching your head. You might encounter references to scripts or files that are no longer available or have been moved. That's exactly the issue we're addressing today â€“ the missing make_image_bootable.sh script. This script is crucial for making your image bootable on OVH's infrastructure, and its absence can be a major stumbling block. In the following sections, we'll explore the purpose of this script and how to achieve the same results even without it.

Step-by-Step Guide to Installing Ubuntu 20.04 on OVH Bare Metal (BYOL)

Okay, let's get down to the actual installation process. We'll break it down into clear, actionable steps. Follow along, and you'll have your Ubuntu 20.04 server up and running in no time!

Step 1 Preparing Your Ubuntu 20.04 Image

The first step involves creating a base Ubuntu 20.04 image. You have a couple of options here: you can either download a minimal Ubuntu ISO and install it in a virtual machine (like VirtualBox or VMware), or you can use a tool like debootstrap to create a minimal system directly. We'll go with the virtual machine approach for this guide, as it's generally more user-friendly, especially if you're not super comfortable with the command line.

  1. Download the Ubuntu 20.04 Minimal ISO: Head over to the Ubuntu website and download the minimal ISO image for the server version. This image is small and contains only the bare essentials, which is perfect for our needs.

  2. Create a Virtual Machine: Open your virtualization software (VirtualBox, VMware, etc.) and create a new virtual machine. Allocate at least 20GB of disk space and 2GB of RAM. For the network settings, choose a bridged adapter if you want your VM to have a direct connection to your network. This is crucial for accessing the VM later.

  3. Install Ubuntu in the VM: Boot the VM from the downloaded ISO image. Follow the on-screen instructions to install Ubuntu. When partitioning the disk, create at least two partitions: one for / (the root filesystem) and one for /boot. A separate /boot partition is often required for BYOL installations on OVH.

  4. Configure the Base System: Once the installation is complete, log in to your new Ubuntu VM. Update the package list and upgrade the installed packages using the following commands:

    sudo apt update
sudo apt upgrade -y

Step 2 Configuring the Network

Network configuration is crucial for your server to communicate with the outside world. OVH uses a specific network configuration, so we need to make sure our image is compatible. This typically involves setting up the correct IP address, gateway, and DNS servers.

  1. Install ifupdown: OVH's rescue system often uses ifupdown for network management, so let's install it:

    sudo apt install ifupdown

  2. Configure Network Interfaces: Edit the /etc/network/interfaces file using your favorite text editor (like nano or vim). You'll need to configure the primary network interface (usually eth0) with a static IP address, gateway, and netmask provided by OVH. You can find these details in your OVH control panel.

    Here's an example configuration. Remember to replace the placeholders with your actual values:

    auto eth0
iface eth0 inet static
address <YOUR_IP_ADDRESS>
netmask <YOUR_NETMASK>
gateway <YOUR_GATEWAY>
dns-nameservers 213.186.33.99 213.186.33.99

  3. Disable systemd-networkd (if necessary): Ubuntu 20.04 uses systemd-networkd by default. In some cases, this can conflict with the ifupdown configuration. If you encounter network issues, try disabling systemd-networkd:

    sudo systemctl disable systemd-networkd
sudo systemctl stop systemd-networkd

Step 3 Installing the Bootloader (Grub)

The bootloader is responsible for loading the operating system kernel. For OVH bare metal servers, we'll typically use Grub. We need to install Grub to the correct disk and configure it to boot our Ubuntu installation.

  1. Install Grub: Install the grub-pc package:

    sudo apt install grub-pc

  2. Identify the Disk: Determine the disk device name where you want to install Grub (e.g., /dev/sda). You can use the lsblk command to list the available disks.

  3. Install Grub to the MBR: Install Grub to the Master Boot Record (MBR) of the disk:

    sudo grub-install /dev/sda

    Replace /dev/sda with the actual disk device name.

  4. Update Grub Configuration: Update the Grub configuration to detect your installed operating system:

    sudo update-grub

Step 4 Making the Image Bootable The make_image_bootable.sh Alternative

This is where things get interesting! Remember the missing make_image_bootable.sh script? This script typically handled several crucial tasks, such as:

  • Installing Grub to the correct location.
  • Generating a grub.cfg file that's compatible with OVH's bootloader.
  • Ensuring the initramfs (initial RAM filesystem) contains the necessary modules for booting on OVH's hardware.

Since we don't have the script, we need to perform these tasks manually. Luckily, we've already covered the Grub installation. Now, let's focus on the other aspects.

  1. Verify Initramfs: Ensure your initramfs contains the necessary modules for your hardware. This is usually handled automatically by update-initramfs, but it's always good to double-check. You might need to add modules for your network card or storage controller.

    sudo update-initramfs -u -k all

  2. Configure Grub for OVH: While the default grub.cfg generated by update-grub usually works, you might need to make some adjustments for OVH's environment. This is particularly important if you're using software RAID or LVM. The key is to ensure that Grub can locate your root filesystem.

    • Check the GRUB_CMDLINE_LINUX_DEFAULT variable in /etc/default/grub. It should contain the correct root device (e.g., root=/dev/sda1 or root=UUID=your-uuid).
    • If you're using software RAID or LVM, you might need to add the mdadm or lvm2 hooks to your /etc/initramfs-tools/modules file.
    • After making any changes, run sudo update-grub and sudo update-initramfs -u -k all to apply them.

Step 5 Creating the Disk Image

Now that we've configured our Ubuntu installation, it's time to create a disk image. We'll use the dd command for this, which is a powerful tool for copying data at a low level.

  1. Shutdown the VM: Shut down your Ubuntu virtual machine.

  2. Locate the Virtual Disk File: Find the virtual disk file for your VM. This is usually a .vdi (VirtualBox) or .vmdk (VMware) file.

  3. Convert the Image (if necessary): OVH typically requires a raw disk image (.raw or .img). If your virtual disk is in a different format, you'll need to convert it. You can use qemu-img for this:

    qemu-img convert -f vdi -O raw <your_disk.vdi> <your_disk.raw>

    Replace <your_disk.vdi> with the actual path to your virtual disk file, and <your_disk.raw> with the desired output filename.

  4. Create the Image using dd: This step is crucial, but also a bit risky, as a mistake can overwrite data. Double-check your commands before running them!

    Important: It's generally safer and faster to create a sparse image. A sparse image only contains the used blocks, making it smaller and quicker to copy. We can achieve this by using qemu-img convert directly to create a qcow2 image (which supports sparseness) and then converting it to raw format with the sparse option enabled.

Hereâ€™s how:

qemu-img convert -O qcow2 <your_disk.vdi> <your_disk.qcow2>
qemu-img convert -f qcow2 -O raw -c <your_disk.qcow2> <your_disk.raw>

The -c option in the second command enables compression, resulting in a sparse raw image.

Alternatively, If you choose to go with the dd command directly (which is generally not recommended for large disks):

```bash
sudo dd if=/dev/sda of=disk.img bs=1M status=progress
```

**_Replace `/dev/sda` with the actual disk device in your VM (if you're doing this from within the VM) or the path to your virtual disk file (if you're doing this from your host machine)._**

**_Be extremely careful with this command! A wrong `if` or `of` can lead to data loss._**

Step 6 Uploading and Installing the Image on OVH

Alright, we've got our disk image. Now it's time to upload it to OVH and install it on our bare metal server.

  1. Access the OVH Control Panel: Log in to your OVH control panel.

  2. Go to your Bare Metal Server: Navigate to the section for your bare metal server.

  3. Enter Rescue Mode: Boot your server into rescue mode. This is a minimal environment that allows you to access the disk and install your image.

  4. Upload the Image: You'll need to upload your disk image to the server. There are several ways to do this, such as using scp or ftp. OVH's rescue system usually provides an SSH server, so scp is often the easiest option.

    scp <your_disk.raw> root@<your_server_ip>:/tmp/disk.raw

    Replace <your_disk.raw> with the path to your disk image, and <your_server_ip> with the IP address of your server in rescue mode.

  5. Install the Image: Once the image is uploaded, log in to your server via SSH (using the rescue mode credentials). Then, use dd again to write the image to your server's disk.

    Again, be extremely careful with this command!

    dd if=/tmp/disk.raw of=/dev/sda bs=1M status=progress

    Replace /dev/sda with the actual disk device on your server.

  6. Reboot the Server: After the image is written, reboot your server from the hard disk. If everything is configured correctly, your Ubuntu 20.04 installation should boot up!

Step 7 Post-Installation Configuration

Once your server is up and running, there are a few post-installation tasks you might want to perform:

  1. Update the System: Update the package list and upgrade the installed packages:

    sudo apt update
sudo apt upgrade -y

  2. Install SSH Key: For security, it's highly recommended to disable password-based SSH login and use SSH keys instead. Generate an SSH key pair on your local machine and copy the public key to your server's ~/.ssh/authorized_keys file.

  3. Configure a Firewall: Install and configure a firewall, such as ufw, to protect your server from unauthorized access.

Additional Tips for OVH Compatibility

Here are a few extra tips to ensure your Ubuntu 20.04 installation works smoothly on OVH:

  • Use a Separate /boot Partition: As mentioned earlier, a separate /boot partition is often required for BYOL installations on OVH.
  • Check the OVH Documentation: Always refer to the latest OVH documentation for any specific requirements or recommendations.
  • Test in Rescue Mode: If you encounter issues, booting into rescue mode can help you troubleshoot and fix them.

FAQ

What if my server doesn't boot after installing the image?

If your server fails to boot, the most likely causes are issues with the bootloader or the initramfs. Boot into rescue mode and check the Grub configuration (/boot/grub/grub.cfg) and the initramfs contents. Make sure the root device is correctly specified and that the necessary modules are included.

How can I troubleshoot network connectivity issues?

If you can't connect to your server after installation, double-check your network configuration in /etc/network/interfaces. Ensure that the IP address, netmask, and gateway are correct. Also, verify that systemd-networkd is disabled if you're using ifupdown.

Can I use this guide for other Linux distributions?

Yes, the general principles outlined in this guide apply to other Linux distributions as well. However, you might need to adjust the commands and configuration files based on the specific distribution.

Where can I get more help?

If you're still stuck, don't hesitate to reach out to the OVH community forums or support. There are many experienced users who can help you troubleshoot your installation.
