Docker In Devuan LXC On Proxmox: A Setup Guide

Aug 8, 2025 by ADMIN 47 views
Iklan Headers

Introduction

Alright guys, let's dive into setting up Docker within an unprivileged Devuan LXC container on Proxmox. This setup is perfect for a home server or homelab where you want to run various applications like Immich and Jellyfin, all neatly tucked away in Docker containers. Running Docker in an LXC container adds an extra layer of security and isolation, which is always a good thing. Plus, using Devuan gives us a systemd-free environment, which some of us prefer for its simplicity and resource efficiency. This guide will walk you through the process step-by-step, ensuring you have a smooth experience. We'll cover everything from creating the LXC container to installing Docker and running your first containerized application. So, grab your favorite beverage, and let's get started!

Why bother with all this complexity? Well, running Docker directly on your Proxmox host might seem simpler, but it can lead to potential security risks and management headaches down the line. By isolating Docker within an LXC container, you're limiting the blast radius if something goes wrong. If a container gets compromised, it won't compromise your entire Proxmox host. Moreover, LXC containers are lightweight and efficient, making them ideal for running multiple applications without significant overhead. Devuan, being systemd-free, further reduces the resource footprint, giving you more room to play with your applications. Using Docker in an unprivileged LXC container is a robust and secure way to manage your applications, especially when dealing with sensitive data or services exposed to the internet. So, it’s a worthwhile investment in the long run.

Before we jump into the practical steps, let's quickly touch on some prerequisites. First, you'll need a Proxmox server up and running. Make sure you have the necessary ISO image of Devuan downloaded. Basic familiarity with Linux command line and Proxmox interface is also expected. Don't worry if you're not an expert; this guide is designed to be beginner-friendly. We'll explain each step in detail, so you can follow along even if you're relatively new to Proxmox and LXC containers. However, having some experience with Docker will be beneficial, especially when it comes to configuring and managing your containerized applications. If you're completely new to Docker, consider checking out some introductory tutorials before proceeding. With these prerequisites in mind, you'll be well-equipped to set up Docker in an unprivileged Devuan LXC container and enjoy the benefits of a secure and efficient homelab environment.

Creating the Unprivileged LXC Container in Proxmox

Okay, first things first, let’s create that LXC container in Proxmox. Log into your Proxmox web interface. Click on “Create CT” in the upper right corner. This will kick off the container creation wizard. Now, enter a name for your container (something descriptive like “devuan-docker”) and set a password for the root user. Pick a strong one, folks! Next, you'll need to select the template. Since we're using Devuan, you might need to upload a Devuan template if it's not already available. Download the appropriate Devuan LXC template from the Devuan website or a trusted mirror. Once you've uploaded the template, select it from the dropdown menu. Configure the disk size, CPU cores, and memory allocation according to your needs. For a basic setup, 20GB of disk space, 2 CPU cores, and 2GB of RAM should suffice. However, if you plan to run resource-intensive applications, consider increasing these values. Finally, configure the network settings. You can either use DHCP to automatically obtain an IP address or set a static IP address. If you choose a static IP, make sure it's within your network range and doesn't conflict with any other devices on your network. Double-check all the settings, and then click “Create”. Proxmox will now create the LXC container based on your specifications.

Once the container is created, don't start it just yet. We need to make a crucial configuration change to ensure it runs as an unprivileged container. Select the newly created container in the Proxmox interface, and then click on “Options”. Look for the “Features” section and double-click on “Nesting”. Enable the nesting feature, which is required for running Docker inside an unprivileged container. Enabling nesting allows the container to run other containers within it, which is exactly what we need for Docker. Next, go to the “Resources” section and check the CPU and Memory limits. Make sure they are set appropriately for your needs. If you plan to run multiple Docker containers within the LXC container, you might want to increase the CPU and memory limits to avoid performance issues. With the nesting feature enabled and the resources configured, you're now ready to start the container. Click on the “Start” button to boot up the Devuan LXC container. After the container has started, you can access it via the Proxmox web interface by clicking on the “Console” button. This will open a terminal session where you can log in as the root user using the password you set during the container creation process. Congratulations, you've successfully created and configured an unprivileged Devuan LXC container in Proxmox!

Now that you're logged into the container, it's time to update the system and install the necessary dependencies. First, update the package list by running apt update. This will refresh the list of available packages from the Devuan repositories. Then, upgrade the installed packages to their latest versions by running apt upgrade. This will ensure that your system is up-to-date with the latest security patches and bug fixes. During the upgrade process, you might be prompted to configure certain packages. Follow the prompts carefully and choose the options that best suit your needs. After the upgrade is complete, install the sudo package by running apt install sudo. This will allow you to execute commands as the root user without having to log in as root directly. Create a new user account with adduser <username>, replacing <username> with your desired username. Follow the prompts to set a password and provide any additional information. Add the new user to the sudo group by running usermod -aG sudo <username>. This will grant the user administrative privileges. Log out of the root user and log in as the newly created user. You can now use sudo to execute commands that require root privileges. This is a more secure way to manage your system than logging in as root directly.
